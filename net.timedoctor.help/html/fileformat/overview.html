TimeDoctor File Format

The default extension of a TimeDoctor trace file is *.tdi. A TimeDoctor trace file consists of a number of lines, each starting with a tag and followed by tag parameters. Since the files are computer generated, the parser is optimized for speed, and is very picky about syntax and file consistency. Any errors in the input file may generate unpredictable results in the viewer.

Constraints and interpretations

This section describes the various constraints and interpretations of the sample sequence in the file in random order:
  • When sampling is started after execution has already started and many task have been created, some artifacts may occur. These are the result of STO samples without corresponding STA samples. TimeDoctor will essentially ignore these STO samples.
  • TimeDoctor assumes that always exactly one task is active per CPU. It is possible, but not recommended to have 0 tasks active. This means that a context switch should always specify a STO <task1> / STA <task2> sequence of samples in the file.
  • Interrupts can be displayed interrupting tasks and other interrupts. Interrupts may even interrupt itself. This means that only for Isr's (not Tasks, see above) nested occurrences of STA/STO pairs may occur.
  • Queue sizes are derived by adding 1 (or the optional size value) for each send and subtracting 1 (or the optional size value) for each receive, limiting at 0. Some startup artifacts may occur when sampling starts when queues are already non-empty. Similar artifacts occur for Semaphores and Queues.
  • DSC tags are always associated with the immediately preceding STA/STO/OCC tags. Any VAL tags in between will not interfere with this association
